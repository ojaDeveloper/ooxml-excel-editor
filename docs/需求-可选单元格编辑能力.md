# 需求文档:为通用 Excel 预览组件增加"可选单元格编辑"等能力

> **组件**:`ooxml-excel-preview`(自研、高保真、只读的通用 .xlsx 预览组件,Vue + React 双框架)。
> **给谁看**:本组件的开发、排期负责人,以及任何评估"能否用本组件做带填写/确认的 Excel 场景"的调用方。
> **一句话**:本组件目前**只能看不能填**。有些使用场景需要在预览的基础上,让**指定单元格可以填写并把值回传**给调用方。本文列出为支持这类场景,组件要补的**通用能力**——并坚持组件永远是中立工具,**不绑定、不写死任何具体业务**。

---

## 一、背景(一张图看懂)

**组件现状**:把一份 .xlsx **漂亮地画出来**(只读)。已支持合并单元格、冻结表头、条件格式、图表、迷你图、数字格式、查找、筛选、导出 PDF、打印等;Vue 和 React 都能用。

**目标场景**:部分调用方需要的不只是"看",而是这样一条链路——

```
   打开一份 Excel  ──▶  组件画出来(像看 Excel)
                          │
                          ├─ 其中某些格子被标记为"可填写"
                          │     文本 ____   数字 ____   日期 ____   下拉[▼]   自定义[…]
                          ├─ 用户在格子里填值
                          └─ 每填一格 → 组件把"哪个格、填了什么"回传给调用方
                                         (存哪、怎么用,由调用方决定)
```

**结论**:要支持这类场景,核心就一件事——**给只读组件加一套"可选的填写能力"**,并配几项相关通用能力。

---

## 二、三条不可动摇的设计原则

1. **默认只读**:编辑等新能力都是"开关"。不打开时,组件的行为与性能和现在**完全一样**,对只看不填的调用方零影响。
2. **不做公式重算**:用户填的值**原样收下**并回传,组件**不**替 Excel 重新计算公式(沿用 Excel 里已存好的结果)。
3. **完全中立、不耦合任何业务**:组件核心不认识任何具体业务概念。调用方"用它来干什么业务",组件一概不知道、也不该出现在组件代码/接口命名里。

---

## 三、要补的通用能力(共 7 条需求)

> 每条按【场景需要什么】→【组件要提供什么】→【打个比方】→【中立红线】来写。

### 需求 1 ⭐ 让指定单元格可填写(通用编辑能力)
- **场景需要什么**:某些格子要能点进去填值,填完保存,并让调用方知道填了什么。
- **组件要提供什么**:支持由调用方声明"哪些格子可编辑、用哪种内置编辑器"——内置**文本 / 数字 / 日期 / 下拉选择**四种常用输入。用户双击格子(或按 F2、或直接打字)进入填写,回车保存、Esc 取消;保存后画面立即显示新值,并对外抛出一个**通用的"单元格已提交"事件**(含:第几张表、行列、新值、旧值)。
- **打个比方**:把一张只读的表,变成在指定位置贴了几张"输入框贴纸",填完会有人替你记下来。
- **中立红线**:由调用方通过通用回调/区域描述声明"哪些格能填、用哪种输入框";组件不关心这格子在业务上意味着什么。事件名用通用词(如 `cell-commit`),不带任何业务含义。

### 需求 2 ⭐ 复杂/特殊输入交给调用方(自定义编辑器扩展点)
- **场景需要什么**:内置四种编辑器不够用时(例如带自动补全的输入、文件/图片上传、特殊的选择器等),需要调用方自己实现的输入控件。
- **组件要提供什么**:编辑器类型支持"自定义"。遇到这种格子,组件只负责**让出位置**——给出一块矩形区域、当前值、以及"保存/取消"两个回调;**控件长什么样、怎么交互,完全由调用方塞进来**(通过 Vue 具名插槽 / React render-prop)。
- **打个比方**:组件挖好一个"画框",调用方想挂什么画自己挂,组件不管画的内容。
- **中立红线**:任何具体的自定义控件实现**一行都不进组件核心**,全部留在调用方。

### 需求 3 可编辑格的"提示外观"(中性、可主题化)
- **场景需要什么**:让用户一眼看出"这些格子要填"。
- **组件要提供什么**:可编辑的格子默认有一个**中性的淡标记**(如灰色虚线框),并**允许通过现有主题机制改成任意颜色/样式**。
- **打个比方**:默认是铅笔淡线;调用方想要别的颜色,调一下主题即可。
- **中立红线**:组件默认不写死任何特定颜色或业务语义,只提供一个中性、可覆盖的标记。

### 需求 4 兼容 WPS 导出的单元格内嵌图片(格式覆盖)
- **场景需要什么**:不少 .xlsx 由 WPS 导出,其单元格内嵌图片用了 WPS 私有存法(DISPIMG),普通解析读不出来,图会缺失。
- **组件要提供什么**:能识别并正常显示这种 WPS 私有的单元格图片。
- **打个比方**:多支持一种"图片的包装方式",否则这类文件里的图显示不出来。
- **中立红线**:这是通用的"文件格式兼容性提升",与任何业务无关,所有此类文件都受益。

### 需求 5 滚动定位到指定单元格(导航 API)
- **场景需要什么**:表很长时,需要把视图定位到某个目标格子并确保它出现在屏幕内。
- **组件要提供什么**:新增命令式 API:"滚动到指定单元格并使其可见"。
- **打个比方**:类似"跳转到第几行",自动把目标格滚进可视区。
- **中立红线**:纯几何导航;"该定位到哪个格"由调用方算好再调用,组件不含任何业务判断。

### 需求 6 冻结窗格下,叠加控件位置不跑偏(裁剪)
- **场景需要什么**:表常冻结表头/首列,滚动时填写框/叠加控件要跟着对应格子走、超出区域要被遮住,不能飘到表头上。
- **组件要提供什么**:把组件内部已实现的"冻结分区(主区/冻结行/冻结列/冻结角)定位裁剪"能力,延伸到调用方挂载的编辑器与叠加层。
- **打个比方**:让"输入框贴纸"也遵守冻结窗格的边界,不越界乱跑。
- **中立红线**:纯显示定位能力,无业务。

### 需求 7 Vue 与 React 双框架对等
- **场景需要什么**:调用方可能用 Vue,也可能用 React。
- **组件要提供什么**:上述新能力(编辑回调、提交事件、自定义编辑器钩子、命令式 API)在 **Vue 壳与 React 壳对等可用、用法一一对应**;底层做成框架无关、双壳共用一套核心。
- **打个比方**:同一套能力出"两个语言版本",功能完全对应。
- **中立红线**:框架无关核心里不得 import 任何具体 UI 框架。

---

## 四、组件不负责的部分(留给调用方)

下列均属于调用方的业务,**组件不提供、不掺和**(此处仅作边界说明,不指向任何具体业务):
- 编辑值的持久化、提交后的状态流转、确认/暂存等动作;
- 任何"自定义编辑器"的具体实现(组件只提供需求 2 的扩展口子);
- 业务侧的导航判断(组件只提供需求 5 的滚动定位,不判断"该去哪")。

---

## 五、三条红线(验收逐条对照)

1. **核心代码不出现任何业务概念**:`src/core/**` 不得 import vue/react,不得出现任何特定业务词,接口/事件命名一律通用。
2. **不开启 = 零影响**:所有新能力均为可选开关;关闭时组件行为与性能和当前只读版**逐字节一致**。
3. **不做公式重算**:编辑仅捕获并回传用户输入的值,不重算公式。

---

## 六、不在本次范围

- 公式重算引擎、撤销/重做、多人协同编辑;
- 重型的"在线 Office 编辑"集成;
- 第四节列出的、属于调用方业务的部分。

---

## 七、分期建议与验收

| 期 | 内容 | 完成后可得到 |
|---|---|---|
| **P1 可填写最小集** | 需求 1 + 2 + 3 + 4 | 调用方可在预览之上**填写并回传**,且文件格式覆盖更全 |
| **P2 体验完善** | 需求 5 + 6 | 长表导航顺畅、冻结窗格下控件不跑偏 |
| **P3 跨框架收口** | 需求 7 + 文档/测试补齐 | Vue/React 对等,组件可对外发布 |

**每期统一验收标准**:
- 第五节三条红线逐条通过;
- 新能力"开 / 关"两种状态都测到;
- 组件原有自动化测试(单元 + 真浏览器)**全绿不破**;
- 涉及对外用法变更的,同步更新文档。

---

## 八、名词小词典(便于非技术阅读)

| 词 | 大白话 |
|---|---|
| 本组件 / 预览组件 | 自研的、能高保真显示 Excel 的通用工具(`ooxml-excel-preview`) |
| 调用方 / 使用方 | 引入并使用这个工具的人(可以是任意业务系统) |
| 只读 | 只能看,不能改 |
| 编辑能力 / 编辑层 | 给组件新增的"能填写"能力 |
| 内置编辑器 | 组件自带的文本/数字/日期/下拉四种输入框 |
| 自定义编辑器 | 内置不够用时,由调用方自己实现、挂进组件预留口子的输入控件 |
| 公式重算 | 改了数据后自动重算公式——**本组件不做** |
| 冻结表头/窗格 | 滚动时让表头/首列固定不动 |
| 框架无关 / Vue·React 双壳 | 同一套能力,Vue 和 React 都能用,共用一套核心 |
