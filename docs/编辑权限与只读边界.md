# 编辑权限与只读边界(2026-06-08, 1.2.0)

把"哪些格能改 / 哪些不能 / 撞墙了发生什么 / 视觉怎么提示"四件事讲清楚。所有 API 都建在唯一的权限闸门 `resolveEditable` 上。

---

## TL;DR

```ts
// 默认: 整个组件只读 (editable: false)
<ExcelViewer :src="file" />

// 开编辑模式: 默认整表可编辑
<ExcelViewer :src="file" editable />

// 白名单模式: 默认只读, 只列出的格 / 行 / 列 / 区域可编辑
<ExcelViewer
  :src="file"
  editable
  :editable-targets="[
    { row: 0, col: 0 },                                  // 单格 A1
    { row: 5, col: 7 },                                  // 单格 H6
    { row: 4 },                                          // 整行 5
    { col: 5 },                                          // 整列 F
    { top: 6, left: 1, bottom: 7, right: 3 },            // 矩形 B7:D8
  ]"
  :read-only-cell-style="true"                           // 只读格灰底
  :strict-dimensions="true"                              // 拖列宽也受白名单约束
  @permission-denied="onDenied"                          // 撞墙提示
/>
```

---

## 1. 权限判定优先级

每次写入 (改值 / 改样式 / 改宽高 / 合并 / 删行 / 嵌图 etc) 都走同一个闸门:

```
┌─ ① editable === false              → 一律只读 ───────────────────┐
│   (整个组件锁死, 历史行为不变)                                       │
├─ ② editableTargets 设了 + 该格未命中  → 只读  (白名单模式)            │
│   "设了" 包括显式 [] (= 全只读)                                    │
├─ ③ readOnlyRanges 命中该格           → 只读  (黑名单, 白名单内仍可黑) │
├─ ④ cellReadOnly(cell, pos) 返 truthy → 只读                       │
└─ ⑤ 其他                              → 可编辑                     ┘
```

`viewer.isCellEditable(row, col)` 是这套规则的入口, 内部就是 `resolveEditable` (位于 [permissions.ts](../src/core/edit/permissions.ts))。

## 2. EditableTarget — 白名单 4 形态

```ts
type EditableTarget =
  | { row: number; col: number }     // 单格
  | { row: number }                  // 整行
  | { col: number }                  // 整列
  | MergeRange                       // 矩形 { top, left, bottom, right }
```

形状自动识别(看带哪些字段);单值 / 数组都接;允许**不相邻**多 target;命中**任一**就算可编辑。

### prop 用法

```vue
<!-- 单值 -->
<ExcelViewer editable :editable-targets="{ row: 0, col: 0 }" />

<!-- 数组 (协同编辑常用) -->
<ExcelViewer editable :editable-targets="[
  { row: 0, col: 0 },     // 标题
  { row: 5, col: 7 },     // 金额
  { col: 4 },             // 整列备注
]" />

<!-- 显式空数组 = 全只读 (区别于不传 undefined = 全可编辑) -->
<ExcelViewer editable :editable-targets="[]" />
```

### 命令式 API

```ts
viewer.setEditableTargets(targets)    // 改运行时白名单, 不动 prop, 立即重绘
viewer.getEditableTargets()           // 读当前白名单 (undefined = 未启用)
viewer.isCellEditable(row, col)       // 该格此刻能不能改
```

## 3. DimTarget — 尺寸 API 3 形态

```ts
type DimTarget =
  | number                              // 单 index
  | number[]                            // 多 index, 允许不相邻
  | { from: number; to: number }        // 闭区间范围
```

### API

```ts
// 设宽高 (多 index 单次 undo)
viewer.setColumnWidth(5, 100)                  // 单值
viewer.setColumnWidth([2, 5, 8], 80)           // 数组 3 列同改
viewer.setColumnWidth({ from: 10, to: 12 }, 60) // 范围 3 列
viewer.setRowHeight([0, 1, 2], 40)             // 多行

// 自动适应 (不传 = 整表)
viewer.autoFitColumns()
viewer.autoFitColumns({ from: 0, to: 4 })
viewer.autoFitRows([3, 7])

// 重置到默认 (移除 Map 条目, 回落 defaultColWidth/RowHeight)
viewer.resetColumnWidth([2, 5])
viewer.resetRowHeight({ from: 0, to: 9 })
```

返回值都是 **`number` 成功条数**(从 1.2.0 起,旧 boolean 升级)。

### strictDimensions 严格模式

默认 `false`: 尺寸 API 仅受全局 `editable` 控制(松)。

设 `true` + 启用了 `editableTargets`:**该列/行至少有 1 格在白名单内才能改尺寸**, 否则 skip + emit `permission-denied: { reason: 'dimension', dims: { axis, indices } }`。跟"白名单未覆盖 = 完全只读"严格语义一致。

```vue
<ExcelViewer
  editable
  :editable-targets="[{ row: 0, col: 0 }]"
  strict-dimensions                              <!-- A1 在白名单 → 第 0 行 / 第 0 列 可改宽高; 其他列/行的尺寸 API 被拒 -->
/>
```

## 4. permission-denied — 撞墙通知

撞墙时**默认静默 skip**(跟现有 `editRange` 一致, 不阻 happy path), 但一次操作结束 emit 一次:

```ts
interface PermissionDeniedPayload {
  reason: 'paste' | 'merge' | 'unmerge' | 'image-place' | 'image-convert' | 'dimension' | 'other'
  cells: Array<{ row: number; col: number }>           // 撞到的格 (per-cell 场景)
  dims?: { axis: 'col' | 'row'; indices: number[] }    // 'dimension' 场景下被拒的列/行
  message?: string                                      // 人类可读说明
}
```

### Vue

```vue
<ExcelViewer
  editable
  :editable-targets="..."
  @permission-denied="onDenied"
/>
<script setup>
function onDenied(p) {
  // 弹 toast / 高亮 / 记日志 / ...
  toast.error(`[${p.reason}] 拒绝 ${p.cells.length} 个格`)
}
</script>
```

### React

```tsx
<ExcelViewer
  editable
  editableTargets={...}
  onPermissionDenied={(p) => toast.error(p.message)}
/>
```

### 覆盖的 mutation 路径(撞只读会发事件的)

| 路径 | reason |
|---|---|
| `pasteRich` (Excel/WPS 复制的 HTML) 合并粘贴 | `paste` |
| `pasteRich` 图片粘贴 | `paste` |
| `mergeCells` / `unmergeCells` | `merge` / `unmerge` |
| `convertImageToCell` / `convertImagesToCells` / `convertImagesInRangeToCell` | `image-convert` |
| `convertCellImageToFloat` / `convertCellImagesToFloats` / `convertCellImagesInRangeToFloat` | `image-convert` |
| `setColumnWidth` / `setRowHeight` / `autoFitColumns` / `autoFitRows` / `resetColumnWidth` / `resetRowHeight` (仅 `strictDimensions=true` 时) | `dimension` |

**不发事件 (旧行为)**:`editCell` / `editRange` / `clearRange` / `setStyle` — 这几个 API 历史上就是"撞只读 silently skip"+ 返 false / 0, 调用方靠返回值判断;Phase A 没改这部分以免破坏老代码。

## 5. 只读视觉(readOnlyCellStyle)

让消费方或插件给只读格画个视觉区分,默认无差异(老行为不变)。

### prop 4 种形态

```vue
<!-- false (默认) = 无视觉差异 -->
<ExcelViewer ... :read-only-cell-style="false" />

<!-- true = 内置浅灰底 #f5f7fa -->
<ExcelViewer ... :read-only-cell-style="true" />

<!-- CellStyleOverride = 固定样式给所有只读格 -->
<ExcelViewer ... :read-only-cell-style="{ fill: { type: 'solid', fgColor: '#fff3cd' } }" />

<!-- CellStyleFn = 按格自定义 (Phase C: cellStyle 升级了第 3 入参) -->
<ExcelViewer ... :read-only-cell-style="(cell, pos, ctx) => ({ font: { italic: true, color: '#999' } })" />
```

### 插件路径(`cellStyle` 钩子升级)

`cellStyle: (cell, pos, ctx?) => CellStyleOverride | void` — Phase C 加了可选第 3 入参 `ctx: { editable: boolean }`。旧 `(cell, pos) =>` 签名兼容。

```ts
const dimReadOnly = definePlugin({
  cellStyle: (cell, pos, ctx) => {
    if (!ctx?.editable) return { font: { color: '#999' } }  // 只读格字变灰
  },
})
```

### 鼠标光标(内置, 不可关)

`editable: true` + 鼠标悬停只读格 → `cursor: not-allowed` 自动。

非编辑模式时整表只读, 不画 not-allowed(整页只读光标会反误导)。

## 6. 完整示例 — 协同编辑场景

```vue
<template>
  <ExcelViewer
    :src="file"
    editable
    :editable-targets="myEditableCells"
    :read-only-cell-style="true"
    strict-dimensions
    @permission-denied="onDenied"
  />
  <toast v-if="deniedMsg" :msg="deniedMsg" />
</template>

<script setup lang="ts">
import { ref } from 'vue'
import type { EditableTarget, PermissionDeniedPayload } from 'ooxml-excel-editor'

// 后端下发: 当前用户能改的格 / 行 / 列
const myEditableCells = ref<EditableTarget[]>([
  { row: 0, col: 0 },                                // 抬头 / 标题
  { top: 4, left: 0, bottom: 100, right: 4 },        // 明细区
  { col: 5 },                                         // 整列备注
])

const deniedMsg = ref('')
function onDenied(p: PermissionDeniedPayload) {
  deniedMsg.value = `🚫 [${p.reason}] ${p.message ?? '部分格不可改'}`
  setTimeout(() => (deniedMsg.value = ''), 3000)
}
</script>
```

## 7. 不应该绕开闸门的迁移指引

如果你写的是 **内部 mutation** (比如自定义插件 / setup 钩子里改格), 推荐:

1. **不要直接动 `sheet.cells.set` / `sheet.columns.set`** —— 改用命令式 API (`viewer.editCell` / `viewer.setColumnWidth` etc), 自动过闸门 + 入命令栈 + 发事件 + 记脏。
2. 命令式 API 返 `false` / `0` 时检查 `viewer.isCellEditable(r, c)` 拿原因, 或听 `@permission-denied` 事件。
3. 必须自己 mutate 时, 先 `if (!viewer.isCellEditable(r, c)) return`。

闸门只在 mutation 入口拦截;**渲染读取**(`getCellValue` / `getCellText` / `inspectCell` 等)永远是允许的, 跟 editable 无关。
